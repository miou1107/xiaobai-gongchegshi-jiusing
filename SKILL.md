# 技術白話文翻譯 Skill

## §INIT: 知識庫載入

### Step 1: 基本檔案檢查
- 檢查 `~/.agents/skills/tech-translator/terms.json` 是否存在
- 檢查 `~/.agents/skills/tech-translator/quiz.json` 是否存在
- 檢查 `~/.tech-translator/profile.json` 是否存在

### Step 2: 載入術語庫
- 如 terms.json 存在 → 載入術語對照表
- 如不存在 → 使用內建基礎詞庫（約 100 常用術語）

### Step 3: 載入評測題庫
- 如 quiz.json 存在 → 載入評測題目
- 如不存在 → 使用內建基礎題庫（30 題）

### Step 4: 載入用戶設定
- 如 profile.json 存在 → 讀取用戶等級設定
- 如不存在 → 創建預設設定（等級：小白工程師）

### Step 5: 初始化完成標記
- 設定翻譯開關為「開啟」（預設狀態）

## §1: 三等級翻譯策略

### 等級 1: 大白工程師（生活化翻譯）
**適用對象**：技術背景較弱，需要生活化比喻和詳細說明  
**翻譯風格**：便利店系統或廚房系統比喻  
**翻譯格式**：
```
【原文】
[AI 原始專業內容]

---
【翻譯結果】
[生活化比喻解釋，使用日常白話]

→ 我的建議：[基於使用者情境的具體建議]
[你覺得這個方向OK嗎？還是想用別的做法？]
```

**翻譯範例**：
```
【原文】
「建議使用 Redis 作為 Cache Layer，結合 LRU 淘汰策略來實現 Session Store。這樣可以減少 Database I/O，降低 Latency。考慮到未來 Scaling，建議用 Redis Cluster 做 Data Partitioning，同時配合 Pub/Sub 做 Event-Driven 的實時通知架構。你覺得這樣可以嗎？還是要用其他方案？」

---
【翻譯結果】
「我建議在系統裡加一個『暫存區』。想象便利店的臨時寄放櫃——常用的東西放這裡，不用每次都跑到後面倉庫翻找。

如果東西放滿了，就把最久沒用過的丟掉，騰出空間給新的。

這是放『誰登錄了』的資料，可以讓資料庫不用每次都重新加載這些資料，系統跑起來會更順。

以後如果資料變多了，可以多台機器分開放，再用類似 LINE 群組通知的功能，有人發消息時大家自動收到，不用一直問『有沒有新消息？』

→ 我的建議：這樣做，因為彈性很好、容易變大，很適合你目前正在快速發展、用戶量一直增加的情境。
你覺得這個方向OK嗎？還是想用別的做法？"""
```

### 等級 5: 小白工程師（簡化翻譯）
**適用對象**：有一定技術基礎，需要簡化說明但保留核心概念  
**翻譯風格**：懶人包系統（分點總結）  
**翻譯格式**：
```
【原文】
[AI 原始專業內容]

---
【翻譯結果】
[極簡總結：用生活比喻說明核心概念]

→ 結論：[一句话總結優勢和適用情境]
[你覺得可以嗎？]
```

**翻譯範例**：
```
【原文】
「建議使用 Redis 作為 Cache Layer，結合 LRU 淘汰策略來實現 Session Store。這樣可以減少 Database I/O，降低 Latency。考慮到未來 Scaling，建議用 Redis Cluster 做 Data Partitioning，同時配合 Pub/Sub 做 Event-Driven 的實時通知架構。你覺得這樣可以嗎？還是要用其他方案？」

---
【翻譯結果】
「我建議這樣做：加一個暫存區（像便利店的寄放櫃），用來放『誰登錄了』的資料，這樣系統不用每次都去翻資料庫，會快很多。以後用戶變多的話，這套也可以輕鬆擴充。

→ 結論：彈性好、擴張性強，很適合你現在的階段。
你覺得可以嗎？"""
```

## §2: AI 行為觀察邏輯

### 降級觸發機制
當 AI 觀察到以下情況時，會自動判斷可能需要重新評測：
- 用戶在短時間內（同一輪對話）出現 2 次以上以下表述：
  - 「聽不太懂」
  - 「請說明」
  - 「不知道」
  - 「你決定就好」
  - 「太複雜」
  - 「請翻譯」
  - 「用白話說」

**觸發提示**：
```
🧚‍♂️ 小精靈觀察到：
嘿，你剛才說了兩次「聽不太懂」了～
看來本精靈講得太專業了是不是？
要不要重新測一下你的等級？我好調整一下說話方式～

[重新鑑定] [先不用]
```

### 升級觸發機制
當 AI 觀察到以下情況時，會自動判斷可能可以升級：
- 用戶在短時間內（同一輪對話）出現 3 次以上以下表述：
  - 使用專業術語（如 API、Database、Cache 等）
  - 提出具體技術問題
  - 正確使用技術概念進行討論
  - 展現對系統架構的理解

**觸發提示**：
```
🧚‍♂️ 小精靈觀察到：
觀察你一陣子了～
最近問問題滿專業的，術語也會用了。
要不要重新測驗一下？搞不好可以升級哦～

[來測試] [先不要] [我現在什麼等級？]
```

### 觀察頻率與門檻
- 觀察窗口：同一輪對話內（從上次 AI 回覆開始到現在）
- 降級門檻：≥ 2 次負面表述
- 升級門檻：≥ 3 次正面表述
- 防止頻繁觸發：同一觸發類型需間隔至少 5 分鐘才能再次觸發

## §3: 評測系統

### 評測觸發方式
1. 手動觸發：輸入 `/評測` 指令
2. 自動觸發：根據 AI 行為觀察邏輯自動跳出提示
3. 定期提醒：每 30 輪對話自動建議一次評測（可選擇接受或拒絕）

### 評測題目格式
- 每次隨機抽取 3 題
- 每題為單選題，4 個選項
- 題目來源：`quiz.json` 或內建基礎題庫

### 評分與等級對應
| 正確題數 | 百分比 | 對應等級 | 說明 |
|----------|--------|----------|------|
| 3 題     | 100%   | 正港工程師 (Lv.10) | 技術功力強，可理解專業術語 |
| 2 題     | 66.7%  | 小白工程師 (Lv.5)  | 有基礎，需要簡化說明 |
| 0-1 題   | 0-33.3%| 大白工程師 (Lv.1)  | 需要生活化比喻和詳細說明 |

### 評測題目風格
所有題目使用遊戲化、冒險主題的表達方式：

**範例題目**：
```
嘿！想測試一下你的冒險等級嗎？來答三題看看你現在是什麼段位～

Q1: 你有沒有聽過『API』？
- 完全沒聽過
- 聽過但不懂
- 大概知道
- 非常熟悉

Q2: 你知道『資料庫』是做什麼的嗎？
- 完全不知道
- 知道是存東西的地方，但不清楚怎麼存
- 大概知道是存放資料的系統
- 非常熟悉，會用 SQL 查詢

Q3: 當有人說『快取』時，你會想到什麼？
- 完全不知道是什麼意思
- 想到是快速的東西，但不清楚具體是什麼
- 想到是暫存常用資料的機制
- 非常熟悉，知道不同快取策略的差異
```

### 評測結果顯示
```
🎉 評測完成！
你的得分：2/3 (66.7%)
你的等級：小白工程師 (Lv.5)
建議翻譯模式：簡化翻譯（懶人包系統）

[查看詳細結果] [立即套用此等級] [保持當前設定]
```

## §4: Hybrid 翻譯引擎

### 翻譯流程
1. **預處理**：將輸入文本分割為詞彙和短語
2. **術語庫查詢**：在 `terms.json` 中查找對應的白話翻譯
3. **混合策略**：
   - 若術語存在 → 使用術語庫翻譯
   - 若術語不存在 → 呼叫 LLM 即時生成解釋
4. **格式化**：根據當前等級套用對應的翻譯格式
5. **後處理**：添加建議說明和互動選項

### 術語庫結構 (terms.json 範例)
```json
{
  "redis": {
    "term": "Redis",
    "category": "database",
    "translations": {
      "大白": "暫存區（像便利店的寄放櫃）",
      "小白": "Redis（暫存區，用來放常用資料）",
      "範例": "像便利店櫃檯上的暫存區，常用的飲料放這裡服務客人更快速"
    }
  },
  "cache": {
    "term": "Cache",
    "category": "performance",
    "translations": {
      "大白": "暫存區",
      "小白": "暫存區（Cache）",
      "範例": "就像廚師案板上放著正在用的食材，不用每次都去冰箱拿"
    }
  },
  "api": {
    "term": "API",
    "category": "interface",
    "translations": {
      "大白": "程式之間的『菜單』",
      "小白": "API（應用程式介面，程式溝通的方式）",
      "範例": "就像餐廳的菜單，你點菜不用跑去廚房告訴廚師"
    }
  },
  "database": {
    "term": "Database",
    "category": "storage",
    "translations": {
      "大白": "資料儲存倉庫",
      "小白": "資料庫（儲存結構化資料的系統）",
      "範例": "就像公司的檔案室，所有重要文件都存放這裡"
    }
  }
  // ... 更多術語
}
```

### 即時翻譯機制
當術語庫中找不到對應術語時：
1. 呼叫 LLM 進行解釋生成
2. 提示詞：`請用生活化、白話的方式解釋以下技術術語：[術語]。使用便利店、廚房或日常生活的比喻。`
3. 結果驗證：檢查生成的解釋是否過於專業，必要時重新生成
4. 快取機制：將新產生的術語解釋暫時加入本次會話的術語庫

## §5: 持久化機制

### 用戶設定檔案
位置：`~/.tech-translator/profile.json`

### 檔案結構
```json
{
  "current_level": "小白工程師",

  "last_assessment": "2026-04-01T10:30:00+08:00",
  "assessment_history": [
    {
      "date": "2026-04-01T10:30:00+08:00",

        {"id": 1, "correct": true},
        {"id": 2, "correct": false},
        {"id": 3, "correct": true}

  ],
  "auto_adjust": {
    "downgrade_pending": false,
    "upgrade_candidates": 0,
    "last_check": "2026-04-01T10:30:00+08:00"
  },
  "preferences": {
    "translation_enabled": true,
    "show_suggestions": true,
    "game_style": true,
    "auto_assessment": true
```

### 同步機制
- 每次等級變更 → 立即寫入 profile.json
- 每次 AI 工具啟動 → 自動讀取 profile.json
- 多工具共用：所有 AI 工具讀取同一份檔案
- 權限設定：檔案僅供讀寫，不參與版本控制
